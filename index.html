Front-End Languages

Front-end development, or client-side development, is all about style and user experience. If you are a creative person and not too great with logic, you probably will want to focus on front-end languages and technologies. Unlike back-end development, your choices of languages are somewhat limited for the front-end. Front-end developers primarily deal with HTML, CSS, and JavaScript. There are some slight deviations from those 3 (I'm talking compiled languages, which we will touch on a bit farther down.), but in the end it all boils down to those 3. Also unlike back-end development, to become a front-end developer you must learn, at the very minimum, at least HTML and CSS. That is just the basics for front-end development though. If you really want to make a career out of front-end development, you had better learn JavaScript as well. Enough with the filler though, let us move on to the languages of the front-end.

HTML

What is HTML?

HTML is the structure of every single website you have ever visited, including this one. Don't believe me? First of all, screw you for calling me a liar!

Second of all, just right click anywhere on this webpage and click on "inspect" or "inspect element". See all those <section>'s and <span>'s? That's HTML! You can go to any website on the internet and do the same thing and you will see those HTML elements and many more.

What HTML is not.

Get ready to be shocked. HTML is not actually a programming language!


I know, right? What am I doing talking to you about a non-programming language in an article about what programming language to learn first for web development? Well, if you want to become a front-end developer (or really a web developer in general) than you have to start with HTML. Instead of being a programming language, it is what is called a markup language and it is a web standard. Meaning if you want to be a web developer, than you have to learn HTML. There is no way around it.

Why you should learn HTML.

Are you even listening to me? I literally just told you that if you want to be a web developer you MUST learn HTML. It is the structure of every website online.

How difficult is HTML to learn?

As far as what you have to learn in order to become a web developer, HTML is by far the easiest. Hell, you could slap together an entire website only using 3 or 4 different HTML elements. It wouldn't be the prettiest or most functional website, but it would be a website.

Where to learn HTML.
  • CodeAcademy - Introduction to HTML
  • FreeCodeCamp - Intro to HTML
  • W3Schools - HTML Path

CSS

What is CSS?

CSS is the style of every website you have ever visited online. If the internet was a house, HTML would be the support and structure, or the 2x4's and drywall. As well as it's cement foundation. Then CSS would be the houses style. It would be the paint, furnishings, wallpaper, and anything else you would consider a visual style. CSS pretties up your HTML and makes it nice to look at.

What CSS is not.

CSS, like HTML, is not actually a programming language.


Instead of being a programming language, it is just a stylesheet or rules for your website. Also, like HTML, CSS is a web standard. Meaning that if you want to be a front-end web developer, than you had better learn CSS.

Why you should learn CSS.

I probably could have just copy and pasted everything from the HTML section here. Anywho, you should learn CSS because, as a front-end developer, you will deal with CSS on a daily (or even hourly) basis. Plus, CSS is fun. There are so many different things you can do with it.

How difficult is CSS to learn?

I could have rated the difficulty of CSS in a couple different ways, but I chose to rate it on how difficult it is to get the basics down. Mastering CSS is a bit more difficult because there is just so much you can do with it.

Where to learn CSS.
  • CodeAcademy - Introduction to CSS
  • FreeCodeCamp - Intro to CSS
  • W3Schools - CSS Path

JavaScript

What is JavaScript?

JavaScript is our first actual programming language! JavaScript gives you the power to make your websites dynamic. It can control, manipulate, and change all of your HTML and CSS content. You can animate your website with JavaScript (You can also animate with CSS, but not to the same degree as with JavaScript.). Just about any website you've ever visited that has an element of interactivity to it has been created with JavaScript. JavaScript is King on the internet. It is one of the most, if not THE most, versatile programming languages around today. Just to keep up with our house analogy for internet technologies, JavaScript would be like the electricity coursing through the wires of the house. So, we have the structure and foundation of the house made from HTML. All the decorations, furnishings, and paint (All the style) of the house is CSS. JavaScript powers it all though. JavaScript makes the house dynamic by giving it parts that can move and change.

Why you should learn JavaScript.

If your goal is to get paid to be a front-end developer, than you had better get good at JavaScript. Sure, you can create some pretty good looking websites with only HTML and CSS, but making websites dynamic and interactive is what is going to get you paid. Back to the house analogy once again. You can certainly build a house with only the structure, foundation (HTML), furnishings, paint, and decorations (CSS). (Just for the record, websites built with only HTML and CSS are called static sites. They are static because they don't have any real moving parts to them) Is anyone going to want to visit that house though? I would think they would want to go to the house with electricity. The house that has air conditioning, heat, and T.V. The house with JavaScript.

How difficult is JavaScript to learn?

JavaScript is considered a fairly easy language to learn and is often recommended as a first language for a beginner to learn. This is because JavaScript is generally pretty forgiving with hoe you write your code. This can be viewed as a positive or a negative to different people. Some developers like how flexible and forgiving JavaScript can be as it makes everything a bit easier. On the other hand, some developers feel like that flexibility and forgiveness can lead to sloppy code. Also, if your writing a large application, it can lead to hard to find bugs in your program by breaking other bits of code down the line that may rely on the sloppy code JavaScript allowed you to write. There are solutions to this problem, some of which we will cover in the compiled (to JavaScript) section.

Compiled (to JavaScript)

What does "Compiled to JavaScript" mean?

Compiled to JavaScript means that you can write your code in a different programming language, and at runtime it will compile your code to JavaScript. Basically it's just a way to write JavaScript without writing JavaScript.

Why not just write JavaScript then?

As we covered in the JavaScript section, many developers don't like how forgiving plain JavaScript can be. A language that is compiled to JavaScript can fix that and make you write cleaner more maintainable code. Languages that compile to JavaScript can also give you different features and benefits such as type-safety and inheritance.

What are some of the languages that compile to JavaScript?
  • TypeScript - The most popular of the compiled to JavaScript languages. Created by Microsoft, TypeScript gives you type-safety and all the newest features from ES6 and ES7.
  • ClojureScript - FUNCTIONAL PROGRAMMING!!!
  • Kotlin - Uses the JVM (Java Virtual Machine). Language of Android development.
  • Elm - Allows you to combine HTML and JavaScript in an easier (and easier to read) way.
  • Dart - Created by Google and the language of the popular mobile app development framework Flutter.

How to write the classic "Hello World" program in 3 of the most famous compiled to JavaScript languages.

  • Dart

    void main() { print("Hello World!"); }

  • Elm

    import Html exposing (text) main = text "Hello World!"

Python

What is Python?

Python is one of the most popular programming languages around today. This is thanks to it's simple syntax, which looks almost like written English. Python, along with JavaScript and Ruby (Which we will get to shortly), is commonly mentioned and recommended as a good first language for a aspiring new programmer to learn. Since Python is not a scripting language, like JavaScript, it has to be used as a back-end language to add logic to your websites.

Why you should learn Python.

As I previously mentioned, Python is among the easiest programming languages to learn for a beginner. It has easy to understand syntax that almost looks like written English. Python also recommended to new programmers because of it's philosophy of "There should be one-- and preferably only one --obvious way to do it." as written in theZen of Python. This philosophy forces new developers to learn the right way of doing something as opposed to many other programming languages which have many ways of doing the same thing.

Some more reasons to choose Python as a first programming language are:
  • It is used by a lot of companies.

    Facebook
    Instagram
    Spotify
    Quora
    Netflix
    Dropbox
    That is just a few of the big companies that use Python a lot in their services. So knowing Python will make you very hireable.
  • The average starting salary for a back-end developer with Python skills is $65,642 dollars (USD) a year.
    According to Payscale.com
  • Since Python is such a popular language, there is a butt load of free resources available online to learn from.
  • Also, since Python is such a popular language, it has a very large and active community online. So, when you get stuck on a problem in your code, it is much easier to get help from someone online than with many other languages.
How difficult is Python to learn?

As I stated earlier, Python is generally considered one of the easiest languages for a beginner to pick up.
